    <h2>Reading a file</h2>

    <h4>Python</h4>
    <pre><code data-trim class="python">
    for line in open("filename"):
        print line
    </code></pre>

    <div class="fragment roll-in">
        <h4>C++</h4>
        <pre><code data-trim class="cpp">
    #include &lt;iostream&gt;
    #include &lt;fstream&gt;
    #include &lt;string&gt;
    using namespace std;
    int main () {
        string line;
        ifstream myfile ("filename");
        if (myfile.is_open()) {
            while ( myfile.good() ) {
                getline(myfile,line);
                cout &lt;&lt; line &lt;&lt; endl;
            }
            myfile.close();
        }
        else cout &lt;&lt; "Unable to open file";
        return 0;
    }
    </code></pre>
    </div>

.. section::

    <h2>Reading a file</h2>
    <h4>Java</h4>
    <pre><code data-trim class="java">
    import java.io.*;
    class FileRead {
        public static void main(String args[]){
            try{
                FileInputStream fstream = new
                FileInputStream("textfile.txt");
                DataInputStream in = new
                DataInputStream(fstream);
                BufferedReader br = new BufferedReader(new InputStreamReader(in));
                String strLine;
                while ((strLine = br.readLine()) != null)   {
                    System.out.println (strLine);
                }
                in.close();
            }
            catch (Exception e){
                System.err.println("Error: " + e.getMessage());
            }
        }
    }
    </code></pre>

    <h2>Why Python ?</h2>
    <img class="fragment roll-out" height="600" src="/images/slides/python/xkcd.png"/><small class="fragment roll-out" style="margin-left: 10px;"> Courtesy: xkcd<sup>[5]</sup></small>

.. section::

    <h2>Why Python ?</h2>
    <ul>
        <li>free and open source</li>
        <li>cross platform</li>
        <li>clear and readable syntax</li>
        <li>rapid application development</li>
        <li>"batteries included" with a great "standard library"</li>
        <li>works great with C, Java, .NET and other technologies</li>
        <li>automatic memory management and garbage collection</li>
    </ul>

.. section::

    <h2>Why Python ?</h2>
    <ul>
        <li>significantly reduces development time and source code size.
            <ul>
                <li>usually 3-5x shorter than equivalent Java programs</li>
                <li>usually 5-10x shorter than equivalent C++ programs</li>
            </ul>
        </li>
        <li>helps the developer focus on code</li>
        <li>excellent language for prototyping</li>
        <li>... and many more</li>
    </ul>

.. section::

    <h2>Why not Python ?</h2>
    <ul>
        <li>often just considered as a scripting language</li>
        <li>slower than compiled languages</li>
        <li>slower parts can often be compiled to C, to make them fast</li>
    </ul>
